About the Diffie-Hellman Method:

Diffie-Hellman is a method in which encryption keys are generated. Is is used in a scenarip where end-to-end encryption
is required, during the creation of the key no information is passed over, only numbers. It is currently unbreakable due
to the way the keys are generated which is explained below.

How it Works:

The two parties come up with two prime numbers, typically will be very large primes (the bigger the prime, the more secure)
and these will be named G and N. Both parties then pick a secret number which can be anything, they keep this to themselves
and don't share it with anyone, these will be names X and Y respectively. They both then perform the calculations below:

Calculations:

Party A performs the calculation of Gx mod N
Party B performs the calculation of Gy mod N

They then pass eachother the results of this calculation and do the following:

Party A performs the calculation of (Party B's result)x mod N
Party B performs the calculation of (Party A's result)y mod N

Party A's end result should equal Party B's end result
And this end result then becomes the encryption key.

This calculation can be represented by the following:
(Gx mod N)y mod N = Gxy mod N
(Gy mod N)x mod N = Gyx mod N

No matter what way the exponentiation is represented you will always get the same answer.
